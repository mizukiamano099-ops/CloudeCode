# 記事作成のための基礎情報ファイル仕様書

このドキュメントは、記事作成時に参照する基礎情報ファイルの作成方法と仕様を説明します。

## 概要

記事作成のための基礎情報ファイルは、記事の品質と一貫性を保つために、執筆前に収集・整理すべき情報をまとめたものです。

## ファイル形式

### 推奨形式

- **主形式**: Markdown (.md)
- **補助形式**: YAML (.yaml) またはJSON (.json) - メタデータ管理用
- **エンコーディング**: UTF-8

### ファイル命名規則

```
{記事タイトル}-reference.md
または
{記事ID}-ref.md

例:
- prompt-engineering-reference.md
- tech-writing-guide-ref.md
- api-documentation-reference.md
```

## 基礎情報ファイルの構成

### 1. 必須セクション

#### 1.1 記事メタデータ

```yaml
難易度: 中級

執筆予定日: 2026-01-18
著者: Claude

ステータス: 執筆中

```

#### 1.2 目的と目標

記事の目的と達成目標を明確にします。

```markdown
## 記事の目的

### 主な目的
この記事は、読者が技術ドキュメントを効果的に作成できるようになることを目指します。

### 達成目標
読者が記事を読み終えた後、以下ができるようになる：
- [ ] 明確で簡潔な技術文書を書ける
- [ ] 読者に適したドキュメント構成を設計できる
- [ ] 効果的なコード例を作成できる
- [ ] ドキュメントの品質をチェックできる
```

#### 2.5 スタイルガイド・トーン

```markdown
## スタイルガイド

### 文体
- **トーン**: フレンドリーだが専門的
- **視点**: 二人称（「あなた」）を使用
- **時制**: 現在形を基本とする

### 文章スタイル
- 1文は50文字以内を目安
- 段落は3-5文程度
- 箇条書きを積極的に活用
- 専門用語には初出時に説明を付ける

### コード規約
- JavaScriptはES6+の構文を使用
- インデント: スペース2つ
- 命名規則: camelCase（変数・関数）、PascalCase（クラス）
- コメント: 日本語で記述

### フォーマット規則
- 見出し: 階層を明確に（H1→H2→H3）
- 強調: **太字**、`コード`、> 引用
- リンク: [説明テキスト](URL)の形式
- コードブロック: 言語指定を必須とする
```

## ベストプラクティス

### 1. 執筆前に基礎情報を完成させる

記事の執筆を始める前に、基礎情報ファイルを80%以上完成させましょう。
これにより：

- 執筆中の迷いが減る
- 一貫性が保たれる
- 必要な情報収集が事前に完了する
- レビュアーが記事の方向性を早期に確認できる

### 2. チームで共有する

基礎情報ファイルは：

- バージョン管理システム（Git）で管理
- 執筆前にレビューを受ける
- チーム内で命名規則を統一
- テンプレートを共有する

### 3. 定期的に更新する

執筆中に気づいたことは随時追加：

- 新たに発見した参考資料
- 執筆中に変更した構成
- レビューでのフィードバック
- 実際の執筆時間の記録

### 4. 記事と一緒に保管する

```
project/
├── articles/
│   ├── tech-writing-guide.md          # 本記事
│   └── tech-writing-guide-reference.md # 基礎情報ファイル
```

## チェックリスト

基礎情報ファイル作成時のチェックリスト：

### 必須項目
- [ ] 記事のメタデータが記入されている
- [ ] 記事の目的が明確に定義されている
- [ ] 対象読者が具体的に記述されている
- [ ] 記事の範囲（含む/含まない）が明記されている
- [ ] 基本的な構成案（アウトライン）がある

### 推奨項目
- [ ] キーコンセプト・用語集がある
- [ ] 参考資料がリストアップされている
- [ ] コード例の準備がある
- [ ] スタイルガイドが定義されている

### オプション項目
- [ ] SEO最適化の計画がある
- [ ] 品質チェック項目がある
- [ ] 更新・保守計画がある

## まとめ

基礎情報ファイルは、記事作成の設計図です。
時間をかけて丁寧に作成することで：

- **執筆効率の向上**: 迷わず執筆できる
- **品質の向上**: 一貫性と完成度が高まる
- **コラボレーション**: チームメンバーとの協力がスムーズ
- **保守性**: 更新時に元の意図を思い出せる

このガイドを参考に、プロジェクトに適した基礎情報ファイルの
フォーマットを確立してください。
