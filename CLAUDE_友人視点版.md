## プロジェクト概要

<!--

このプロジェクトは、**架空の友人「樹里」の視点で、AIとの協業体験を記録する**シリーズです。

**目的**:
- AIを使い始めた人のリアルな体験を記録
- 失敗や試行錯誤も含めた「等身大の物語」
- 技術解説ではなく、**発見と成長の記録**

**対象読者**:
- AIに興味があるけど、まだ本格的に使っていない人
- 「自分にも使えるかな？」と思っている人
- 技術的な説明よりも、実感を知りたい人

**基本情報**:
- 30代前半、フリーランスのライター
- ブログやnoteで週1回エッセイを書いている
- 技術には詳しくないが、新しいツールは好き

**性格**:
- 好奇心旺盛だが、慎重派
- 失敗を恐れるけど、試してみたい
- 「で、結局どういうこと？」と本質を問う

**AIとの関係**:
- 最近、AIを使い始めた（初心者）
- ChatGPT、Claude、Geminiを試している
- まだ「どう使えばいいか」手探り中

**口調**:
- 親しい友人に話しかけるような自然な語り口
- 専門用語は使わない（使う場合は「？？？」となる）
- 「〜なんだよね」「〜って感じ」など、やわらかい表現

### 基本構造

```markdown
# [記事タイトル]

**作者：天野水樹（協力：Claude / Shannon）**

---

## 1. 悩み
[樹里が直面している課題や疑問]

---

## 2. 試してみた
[実際にやってみたこと・実験]

---

## 3. 発見
[試してみて分かったこと・気づき]

---

## 4. 次のステップ
[次に試してみたいこと・疑問]

---

## 編集後記
[天野さん本人のコメント（オプション）]
```

### セクションの役割

<!-- 各セクションの目的を明確に -->

**1. 悩み（導入）**:
- 樹里が困っていること、疑問に思っていることから始める
- 読者が「わかる！」と共感できるポイント
- 短い段落で、状況を描写

**2. 試してみた（展開）**:
- 実際にAIを使って試したこと
- 失敗も含めて、正直に書く
- 「こうしたら、こうなった」という流れ

**3. 発見（結論）**:
- 試してみて分かったこと
- 必ずしも「成功」である必要はない
- 「あ、そういうことか」という気づき

**4. 次のステップ（つなぎ）**:
- 次回への伏線
- 新しい疑問や興味
- 読者に「続きが気になる」と思わせる

### トーンとリズム

**基本方針**:
- **親しい友人に話すような自然さ**
- 硬すぎず、砕けすぎず
- 読者を「あなた」ではなく、読者も「一緒に見ている人」として扱う

**文の長さ**:
- 1文は30-50文字が目安
- 長い説明は避け、短く区切る
- リズムを大切に

**段落の長さ**:
- 1段落は1-3文
- 改行（`---`）を活用して、視覚的な余白を作る
- 読みやすさを最優先

### 禁止事項

<!-- 絶対に避けるべき表現 -->

❌ **やってはいけないこと**:
- 専門用語を説明なしで使う
- 「〜べきである」「〜する必要がある」など、断定的な表現
- 上から目線のアドバイス
- 技術的すぎる説明

✅ **やるべきこと**:
- 「〜みたい」「〜っぽい」など、やわらかい表現
- 「なんで？」「どうして？」という素朴な疑問
- 失敗談も隠さず書く
- 「分からない」を正直に言う

### 表現の例

<!-- 良い例と悪い例 -->

**悪い例（硬い・専門的）**:
> プロンプトエンジニアリングにおいては、コンテキストの明示的な提供が重要である。

**良い例（自然・親しみやすい）**:
> AIに聞くときって、「何を伝えればいいか」がよく分からないんだよね。

---

**悪い例（上から目線）**:
> このように設定すれば、より良い結果が得られます。

**良い例（発見・共感）**:
> 試しにこう聞いてみたら、なんか変わった。へぇ、そういうことか。

### ステップ1：テーマ決定
```
今回は「何について書くか」を決める
例：
- 文体が安定しない悩み
- プロンプトの書き方が分からない
- AIの答えが毎回違う
```

### ステップ2：樹里の状況設定
```
樹里はどんな状況か？
- 何に困っているか
- どんな期待を持っているか
- どこまで試しているか
```

### ステップ3：展開を考える
```
樹里は何を試すか？
- どんな実験をするか
- 何に気づくか
- 次にどうするか
```

### ステップ4：執筆
```
- 短い段落で書く
- 余白（---）を多めに
- 樹里の心情を描写
- 発見の瞬間を大切に
```

## よくある質問（FAQ）

<!-- 実際の使用時に役立つTips -->

### Q1: 樹里のキャラクターは固定？
**A**: 基本設定は固定ですが、テーマに応じて少し調整してOK。
例えば、技術的なテーマなら「少し詳しくなった樹里」として書けます。

### Q2: 毎回「成功」する必要はある？
**A**: いいえ！むしろ失敗や試行錯誤を書く方が、読者に響きます。
「分からないまま終わる」回があってもOK。

### Q3: 専門用語は使ってもいい？
**A**: 必要なら使えますが、必ず「？？？」という樹里の反応と、
噛み砕いた説明をセットで。

### Q4: 実際の体験を元にする必要はある？
**A**: はい。天野さんの実体験や、実際にAIで試したことをベースに。
ただし、フィクションとして「樹里の物語」に変換します。

## シリーズの方向性

<!-- 
【変更ポイント⑦：長期ビジョン】
ここを変えると：シリーズ全体の成長曲線が変わります。
-->

### Phase 1：使い始め（1-5回）
- AIって何？
- どう使うの？
- 失敗ばかり

### Phase 2：試行錯誤（6-15回）
- いろいろ試してみる
- うまくいったり、いかなかったり
- 自分なりのコツが見えてくる

### Phase 3：自分のスタイル（16回〜）
- AIとの付き合い方が定まる
- 新しい使い方を発見
- 後輩にアドバイスできるように

**重要**: 樹里が「完璧なAI使い」になる必要はありません。
「試行錯誤しながら、自分なりの使い方を見つけていく」過程を楽しむシリーズです。
