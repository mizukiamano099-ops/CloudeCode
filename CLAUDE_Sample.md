# CloudeCode - 記事テンプレートプロジェクト

## プロジェクト概要

このリポジトリは、Claude Codeを使用した記事作成のためのテンプレートとサンプルを管理するプロジェクトです。技術記事や技術ドキュメントを効率的かつ一貫性を持って作成するためのガイドラインとテンプレートを提供します。

## プロジェクトの目的

- 高品質な技術記事を効率的に作成するためのテンプレート提供
- 記事作成前の基礎情報整理のための仕様定義
- 一貫性のある記事フォーマットの維持
- ヘッダー・フッター付きの構造化されたドキュメント作成の促進

## ファイル構成

```
CloudeCode/
├── CLAUDE.md                      # このファイル - プロジェクト説明
├── article-template-sample.md     # ヘッダー・フッター付き記事テンプレート
├── article-reference-spec.md      # 記事作成のための基礎情報ファイル仕様書
├── prompt-basics.md               # サンプル記事: プロンプトエンジニアリング基礎
└── test.md                        # テストファイル（空）
```

### 各ファイルの説明

#### article-template-sample.md
**用途**: 新しい記事を作成する際のテンプレート

**特徴**:
- ヘッダーセクション（メタデータ、目次）
- 構造化された本文セクション
- 充実したフッターセクション（関連記事、参考資料、ライセンス、更新履歴）

**使い方**: このファイルをコピーして、内容を書き換えて新しい記事を作成

#### article-reference-spec.md
**用途**: 記事作成前の基礎情報を整理するためのガイド

**特徴**:
- 記事のメタデータ定義
- 対象読者とスコープの明確化
- 参考資料の整理
- 記事構成案の作成
- スタイルガイドの定義

**使い方**: 新しい記事を書く前に、このガイドに従って`{記事名}-reference.md`を作成

#### prompt-basics.md
**用途**: 実際の記事サンプル

**特徴**:
- プロンプトエンジニアリングの基礎を解説
- 完成した記事の例として参照可能

## 記事作成ワークフロー

### 1. 企画段階
```bash
# 基礎情報ファイルを作成
cp article-reference-spec.md my-new-article-reference.md
# 基礎情報を編集・記入
```

### 2. 執筆段階
```bash
# テンプレートをコピー
cp article-template-sample.md my-new-article.md
# 基礎情報を参照しながら記事を執筆
```

### 3. レビュー・公開
```bash
# 変更をコミット
git add my-new-article.md my-new-article-reference.md
git commit -m "Add: [記事タイトル]"
git push
```

## コーディング規約・文書規約

### Markdown スタイル

#### 見出し
- H1 (`#`) は記事タイトルのみに使用
- H2 (`##`) は主要セクション
- H3 (`###`) はサブセクション
- 見出しの前後には空行を入れる

#### コードブロック
- 必ず言語指定を付ける
```markdown
\`\`\`javascript
const example = "code";
\`\`\`
```

#### リスト
- 箇条書きは `-` を使用
- 番号付きリストは `1.` から開始
- ネストする場合はインデント2スペース

#### リンク
- 相対パスを使用: `[リンク](./file.md)`
- 外部リンクは絶対URL

### 文章スタイル

#### トーン
- **フレンドリーで専門的** なトーン
- 読者を「あなた」と呼ぶ（二人称）
- 丁寧語を使用

#### 文章構成
- 1文は50-70文字以内を目安
- 段落は3-5文程度
- 専門用語は初出時に説明

#### 日本語表記
- 漢字とひらがなのバランスに注意
- 「〜」は全角チルダ
- 英数字は半角
- カッコは全角（）を使用

## ファイル命名規則

### 記事ファイル
- 形式: `{トピック}-{サブトピック}.md`
- 小文字とハイフンを使用
- 例: `prompt-basics.md`, `api-documentation-guide.md`

### 基礎情報ファイル
- 形式: `{記事名}-reference.md`
- 記事ファイルと対応させる
- 例: `prompt-basics-reference.md`

### ブランチ命名
- 形式: `claude/{記事トピック}-{セッションID}`
- 例: `claude/article-template-sample-GxcCr`

## Git ワークフロー

### ブランチ戦略
- **main**: 公開済みの記事
- **claude/\***: 記事作成・編集用の作業ブランチ

### コミットメッセージ
```
[種類]: [記事タイトル/変更内容]

詳細説明（オプション）
```

**種類の例**:
- `Add`: 新規記事追加
- `Update`: 既存記事の更新
- `Fix`: 誤字・誤り修正
- `Refactor`: 構成変更
- `Docs`: ドキュメント関連

**例**:
```
Add: プロンプトエンジニアリング基礎記事

プロンプトの書き方、ベストプラクティス、実践例を含む
包括的な入門記事を追加
```

## プロジェクト固有のルール

### やるべきこと ✅

1. **記事作成前に基礎情報ファイルを作成**
   - 目的と対象読者を明確にする
   - 構成案を練る
   - 参考資料を集める

2. **テンプレートを活用**
   - `article-template-sample.md` をベースにする
   - ヘッダー・フッターを必ず含める

3. **一貫性を保つ**
   - 既存記事のスタイルを参考にする
   - 用語を統一する

4. **メタデータを記入**
   - タイトル、カテゴリー、難易度、日付を明記
   - 読了時間を見積もる

### やってはいけないこと ❌

1. **テンプレートを無視しない**
   - 独自のフォーマットで記事を作成しない
   - ヘッダー・フッターを省略しない

2. **基礎情報なしで執筆を始めない**
   - 行き当たりばったりの執筆を避ける
   - 対象読者不明のまま書かない

3. **mainブランチに直接コミットしない**
   - 必ず作業ブランチ（claude/*）を使用
   - レビューなしでマージしない

4. **コード例を検証せずに掲載しない**
   - 動作しないコードを載せない
   - 古い構文を使わない

## よくある質問（FAQ）

### Q1: 新しい記事を作成する手順は？
```bash
# 1. 基礎情報ファイルを作成
cp article-reference-spec.md my-topic-reference.md
# 編集して記事の計画を立てる

# 2. 記事本体を作成
cp article-template-sample.md my-topic.md
# 基礎情報を参照しながら執筆

# 3. コミットしてプッシュ
git add my-topic.md my-topic-reference.md
git commit -m "Add: マイトピックに関する記事"
git push
```

### Q2: 既存の記事を更新する場合は？
```bash
# 記事を編集後
git add [ファイル名].md
git commit -m "Update: [変更内容]"
git push
```

### Q3: ヘッダー・フッターは必須？
はい、必須です。`article-template-sample.md`に従って必ず含めてください。
- ヘッダー: メタデータと目次で読者をナビゲート
- フッター: 関連情報と信頼性を提供

### Q4: コード例はどの言語で書くべき？
記事のトピックに応じて適切な言語を選択してください。
ただし、以下の原則を守ってください：
- 必ず言語指定を付ける
- 動作確認済みのコードのみ掲載
- コメントで説明を追加
